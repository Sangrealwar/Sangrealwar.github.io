原理

常见问题

缓存穿透

本来查询时，是集中在缓存，但缓存失效时，会查询到数据库，如果是热点访问，会给数据库带来非预期的压力。

解决方案：

  • 前置拦截过滤
  • 如果DB数据为空，Mock一个缓存（但需要注意缓存一致性，有效期较短）
  • 增加布隆过滤器

缓存击穿，在缓存穿透的基础上，增加一个突然某个时间点，容易导致下游数据库压力剧增

解决方案：

  • 缓存失效时增加互斥锁
  • 数据永不过期

缓存雪崩

当某一时间突然大量缓存过期，导致下游数据库压力剧增。

解决方案：

  • 随机过期时间
  • 保持Redis稳定性

过期策略

定时过期

针对Key设置过期时间，内存友好，CPU不友好，容易影响吞吐量

惰性过期

当访问Key时，判断Key是否过期，对CPU友好，但内存过期慢。

定期过期

每个一定时间，扫描一定数量的过期Key，可以指定扫描的间隔，以及每一次扫描的限定耗时。

内存淘汰策略

8种策略

从过期的Key中检索 从所有Key中检索
LRU（最近最少使用） volatile-lru allkeys-lru
LFU（最近最久使用） volatile-lfu allkeys-lfu
随机 volatile-random allkeys-random
优先删除剩余时间短的 volatile-ttl
拒绝操作 noeviction（默认）
  • LRU（Least Recently Used，最近最久未使用）

侧重点在于时间，当数据被访问时，加入链表头部，新数据同样放链表头部。尾部的数据就是最近最久未访问的淘汰数据。

  • LFU（Least Frequently Used，最近最少未使用）

侧重点在于访问频次。根据访问次数排序，当数据被访问时，向前排序，可采用采用小顶堆，插入，删除，访问数据操作复杂度（Ologn）

如果采用LRU，当访问量很高的数据，最近没有访问时，会被淘汰掉，因此不太适合热点数据，热点数据主要是指访问频次高的。

使用场景

缓存

Redisson

主要利用String

限流/令牌桶

Sentinel，Redisson

主要利用Hash和zset，作为HashKey存储配置，zset根据时间戳评分检索（使用zrangebyscore）。统计令牌桶数量，令牌桶是滑动窗口。

分布式锁

Redisson

主要利用String（SetNx，Get），可重入的主要利用Hash结构（hincrby，pexpire），根据线程ID作为重入标识。

消息队列

Redisson

入队：利用List，a) 列表里有数据时，lindex定位到位置，linsert插入到指定位置前/后，b)rpush插入到空列表里

出队：lpop

注：因为在入队、出队时，会加锁。因为看门狗机制，会定时续期。

布隆过滤器

Redisson

String（setbit）

数据结构

String

较为常用的数据结构，Value最大范围可以达到512MB，但是工作中使用考虑到使用效率问题，一般参考阿里巴巴的开发手册要求，认为超过1M的数据可认为是大数据

常用命令

Set/get

设置/获取key中的值，不支持直接设置过期时间

Setnx

是”SET if Not eXists”的简写，原子命令，如果存在则不会更新

Setex

设置值的同时设置过期时间，类似的有PsetEx

IncrBy/decrBy

增加/减少，要求value的值是可以转化为Integer类型，如果不能正确转化，则会报错

Mget/Mset/Msetnx

批量获取/批量设置

Getset/GetDel/GetEx

Get操作后附带的操作，其中，GetSet是1.0版本就有的，而GetDel/GetEx需要到6.2之后

其他命令参考文档：https://redis.io/commands/append

特点

作为最常用的Redis命令，可以原子设置过期时间，避免缓存-数据库一致性这个问题，常用于验证码缓存等短期缓存场景。

list

常用命令

Lpush/Lpush，Lpop/Lpop

从左侧/从右侧插入，相对的是从左侧/从右侧弹出

Lrange

检索在一定范围内的元素

Ltrim

截取一定范围内的元素

其他命令参考：https://redis.io/commands/blmove

特点

底层是链表结构，消费者模型可用于队列，可方便的获取最新一次的数据。

hash

常用命令

Hset/Hget，Hmset/Hmget

设置/获取，批量设置/批量获取。

Hincrby/Hdecrby

增加/减少某个key的值

Hsetnx

类似煜setnx

其他命令参考：https://redis.io/commands/hdel

特点

哈希结构，可以比较方便的表示对象内的字段关系，hash里面的key没有独立的过期时间，过期时间跟着大的Key而定。

set

常用命令

Sadd

添加某个元素

Sismember

判断某个元素是否存在，1表示存在

Smembers

返回元素长度

Sunionstore/Sinter/Sdiff

取并集/交集/差集

其他命令参考：https://redis.io/commands/sadd

特点

集合，无排序，可排除重复项。利用并交差，可以快速检索出两个列表的关系，例如关注了同一个人的用户。

sort set（zset）

常用命令

Zadd

增加元素

Zrange

检索指定评分的元素，可以降序

Zrank

查看某个元素属于什么索引（不是评分）

Zinter/Zunion/Zdiff

于set的命令相似

其他命令参考：https://redis.io/commands/bzmpop

特点

set升级版，增加了score，可以根据评分进行排序，底层利用skiplist。常用于排行榜。

bitmap

常用命令

Setbit/Getbit

设置/获取。设置的偏移量必须是Integer类型。

Bitcount

查看位数为1的个数

Bitop

对string进行位运算

Bitpos

查找指定0/1的第一位

其他命令参考：https://redis.io/commands/setbit

特点

位图，是一种特殊的String，可以标识2^32个bi位，可以通过高效的存储结构，利用少量的空间存储大量的数据。例如，布隆过滤器，可以基于bitmap来实现，使用时通过某种hash函数，将数据转化为位表示。也可以用于记录签到，打卡等一次性开关活动。

hyperLogLogs

常用命令

Pfadd

增加元素

Pfcount

返回有多少基数

其他命令参考：https://redis.io/commands/pfadd

特点

也是一种特殊的String，在允许误差的情况下，快速计算基数（也就是不重复的元素个数），常用的数据结构有（set，bitmap）
